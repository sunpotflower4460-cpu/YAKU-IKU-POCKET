# データソースとライセンス（DATA_SOURCES_AND_LICENSES）

> 統合仕様書 §18 準拠。**実データ連携は PR14**。本書は候補と方針のアウトライン。

## 候補ソース
| ソース | 用途 | 種別 |
|---|---|---|
| Pl@ntNet API | 複数画像・部位・候補識別 | 識別 |
| GBIF | taxonomy / occurrence / distribution | 分類・分布 |
| POWO / Kew | taxonomy / traits / distribution / uses | 分類・利用 |
| iNaturalist | 周辺観察・画像・コミュニティ | 観察・画像 |
| YList | 和名⇔学名 | 分類 |
| 厚生労働省 | 有毒植物・食中毒 | 安全 |
| 消費者庁 | 自然毒 | 安全 |
| 自治体/植物園/大学 | 地域フロラ | 分布 |
| IUCN 等 | 保全状態 | 保全 |

## 方針（§18 注意）
- 利用規約・ライセンスを**データ種別ごと**に確認して採用。
- 画像ライセンスを記録し、出典のない画像は保存しない。
- **APIキーをクライアントに置かない**（バックエンド経由）。無断スクレイピング禁止。
- キャッシュ期間・情報更新日を保存。公式データとユーザー投稿を同列表示しない。
- 伝統利用と医学的根拠を分ける。

## PR29: 危険植物14種への実出典の紐付け

このセッション環境ではPR6〜28まで「外部APIキーが無い」「バックエンド未構築」という制約のもと`taxonIds`/`sourceRefs`を全件空にしてきたが、実際には`WebSearch`ツールが利用可能であることが判明した。これはPR14が判断保留とした「実データ連携」（Pl@ntNet等のAPIをアプリのバックエンドに組み込む）とは別の話——アプリの識別ロジックにAPIを組み込むのではなく、**開発時にこのツールで実在する出典を調べて`PlantDefinition.sourceRefs`に記録する**という、静的コンテンツの充実作業として実施した。

- **対象**: `danger: 'RED'`（`safety.level === 'high_risk'`）の14種すべて（トリカブト・スズラン・ハシリドコロ・ドクゼリ・スイセン・イヌサフラン・バイケイソウ類・チョウセンアサガオ・ヨウシュヤマゴボウ・テンナンショウ類（マムシグサ）・ドクニンジン・タケニグサ・ドクウツギ・キョウチクトウ）
- **手法**: `WebSearch`で「植物名＋中毒＋厚生労働省／消費者庁／自治体」等のクエリを実行し、検索結果のタイトルが当該種を明示的に指しているURLのみを採用。厚生労働省「自然毒のリスクプロファイル：高等植物」シリーズを優先し、無い場合は東京都・新潟市等の自治体「食品衛生の窓」ページ、国立医薬品食品衛生研究所(NIHS)のPDFで代替した
- **検証の限界（正直な記載）**: この環境の`WebFetch`ツールはネットワークポリシーにより`.go.jp`/`.lg.jp`ドメインへの接続が拒否される（プロキシから403）。そのため、各URLの本文を逐語的に取得・確認することはできなかった。検証は検索結果自体のタイトル・スニペットが当該種を明示していることの確認にとどまる。アプリ内の植物解説文（`description`/`warningNote`等）はこれらのページからの転記ではなく、従来どおり編集部の一般的な植物学知識をもとに執筆したものであり、出典URLは「詳しくはこちらの公的機関のページを参照」という参照リンクとして添付している
- **反映箇所**: `src/data/plantDefinitions.ts`の`SOURCE_REFS_OVERRIDE`（`PlantDefinition.sourceRefs`と`safety.sourceRefs`の両方に反映）。`reviewStatus`は`'editorial'`のまま変更していない（AIが出典を検索して添付しただけであり、資格を持つ専門家によるレビューではないため、`'expert'`への格上げは行わない）
- **テスト**: `plantDefinitions.test.ts`に、非空の`sourceRefs`はHTTPSかつ`.go.jp`/`.lg.jp`ドメインのみである（＝架空URLや個人ブログ等を紛れ込ませない）ことを検証するテストを追加。RED以外の種の`sourceRefs`は引き続き空であることも検証
- **UI**: 植物詳細画面の「深く学ぶ」タブの「データの確度・出典」欄に、出典があるページのみ実際にタップして開けるリンクを表示（`Linking.openURL`）

## PR30: YELLOW（要注意）植物への出典拡大——正直なカバレッジの記録

PR29の手法をYELLOW（`safety.level === 'caution'`）27種にも適用したが、RED14種と違い「厚生労働省 自然毒のリスクプロファイル」のような専用シリーズページが存在するとは限らず、検索結果の多くが個人ブログ・アフィリエイトサイト等の非公式情報源だった。

- **見つかった5件のみ追加**: ワラビ（食品安全委員会「ワラビと発がん物質」リスク情報）、スターアニス（東京都健康安全研究センター「トウシキミ（八角）とシキミ」— 混同による誤食注意）、モロヘイヤ（農林水産省「モロヘイヤの種には毒があると聞いたが、本当ですか」Q&A）、ナツメグ（日本救急医学会関東地方会雑誌の中毒症例報告、J-STAGE＝科学技術振興機構が運営する政府系論文プラットフォーム上で公開）、センナ（医薬品医療機器総合機構(PMDA)のOTC医薬品添付文書PDF）
- **残り22種は意図的に出典なしのまま**: スギナ・ニワトコ・ホトケノザ・ギシギシ・スイバ・ムラサキケマン・キツネノボタン・カラスビシャク・ヤマウルシ・フジ・セージ・シナモン・クローブ・サフラン・エルダーフラワー・ネトル・ボリジ・ヒソップ・ジュニパーベリー・ウッドラフについては、`.go.jp`/`.lg.jp`ドメインで種名を明示的に扱う十分に具体的なページを発見できなかった。同トピックを扱う一般論的な政府ページ（例:「有毒植物による食中毒に注意しましょう」という総合案内ページ）や、個人の植物図鑑・アロマテラピーサイトはヒットしたが、これらを「その種固有の公的出典」として添付することは、出典の信頼性を実態以上に見せることになるため見送った。**架空の出典を作らないのと同じ理由で、根拠の弱い出典も追加しない**という基準を貫いている
- **反映・検証**: PR29と同じ`SOURCE_REFS_OVERRIDE`・同じテスト基準（`.go.jp`/`.lg.jp`のHTTPS URLのみ）を`caution`レベルにも適用。`plantDefinitions.test.ts`は「`general_observation`(GREEN)のみ常に空」という条件に緩和し、RED/YELLOWいずれも非空なら同じ検証を通す

## PR32: PlantUse（暮らしに活かす）コンテンツへの実出典反映

PR22は「確認済みの調理法データが無い」という理由で全GREEN種に「準備中」の食カードのみを出していたが、`WebSearch`が使えることを踏まえ、農林水産省の郷土料理データベース「うちの郷土料理～次世代に伝えたい大切な味～」（`https://www.maff.go.jp/j/keikaku/syokubunka/k_ryouri/`）で実在する具体的な料理名が見つかった種にのみ、実際の料理名と出典を反映した。

- **見つかった2件のみ追加**: ヨモギ（p002、栃木県の郷土料理「草餅」）、ゴボウ（h040、東京都の郷土料理「きんぴら」）。いずれも`https://www.maff.go.jp/...`の実在する個別ページを`sourceRefs`として`src/data/plantUses.ts`の`FOOD_USE_OVERRIDE`に追加し、`evidenceLevel`を`'official_guidance'`に格上げした
- **他の多くのGREEN種は意図的に「準備中」のまま**: ミツバ・シソ・タラノキ・ミョウガ・ドクダミ等について`WebSearch`を試みたが、農林水産省の郷土料理データベースに個別の該当ページを発見できなかった（一般的なレシピサイト・農産物紹介ページはヒットしたが、政府の公式データベースにおける specific な該当項目ではなかった）。根拠の弱い出典を追加しない方針を継続している
- **食薬区分の誤りを発見・修正**: 調査の過程で、ゲンノショウコ（p031、GREEN＝毒性は無いが）は日本の食薬区分（厚生労働省が運用する食品・医薬品の境界に関する制度）上、地上部が食品ではなく生薬（医薬品的な原材料）として扱われるとする複数の薬用植物園・生薬関連情報源が見つかった。これまでこのアプリは`danger: 'GREEN'`の全種に自動で「料理に使う」カード（「準備中」表示）を出していたため、法制度上は食品として扱えないこの種にも将来レシピが追加され得るかのような誤解を与えていた。`MEDICINAL_NOT_FOOD`という別枠の判定テーブルを設け、該当種のみ「食品ではなく生薬として扱われている」という正確な内容にカードの中身を差し替えた（`plantUses.ts`）。この情報も検索結果のスニペットレベルの確認にとどまるため、厚生労働省の食薬区分制度そのものを説明する公式ページ（`https://www.mhlw.go.jp/stf/seisakunitsuite/bunya/kenkou_iryou/shokuhin/syokuten/iyakuhin/index.html`）を出典として添付し、断定的になりすぎないよう「〜として扱われています」という表現にとどめた
- **検証の限界**: PR29/30と同じ理由（`WebFetch`が`.go.jp`をブロックされる）で、出典ページ本文の逐語確認はできていない
