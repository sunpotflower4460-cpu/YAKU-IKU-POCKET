# 安全ポリシー（SAFETY_POLICY）

> 対象: 薬育ポケット / 統合仕様書 §9.2D, §10.4, §21 準拠。PR6で確立。

## 原則
本アプリは **「食べられるか判定する道具」ではない**。植物を観察し、危険や類似種を学ぶための参考情報を提供する。安全性は「食用可否の断定」ではなく「危険回避の知識」として提供する。

## 実装済みルール（PR6）
1. **本番でランダム植物を返さない**：実AIが特定不能/低信頼/DB外/失敗のときは `unidentified` / `error` を返し、ランダムな代替を出さない（`src/utils/claudeAI.ts`, `src/utils/aiRecognition.ts`）。
2. **判定不能を正式状態化**：`IdentificationState`（`src/types/observation.ts`）。UIは断定しない。
3. **デモ（モック）と本番の完全分離**：`IS_DEMO_MODE`（`src/utils/appMode.ts`）。デモ結果は閲覧のみで、**XP・観察履歴・図鑑に一切反映しない**（`app/(tabs)/scan.tsx`, `ScanResultModal`）。
4. **危険類似種の警告**：LLMではなくデータ（`src/data/safety.ts`）を真実源に、`getSafetyWarnings()` が危険な類似種を返し、`SafetyBanner` が結果画面・詳細画面に色以外（アイコン+見出し+本文）で表示。
5. **禁止表現の撤去**：`食用可 / 食用可能 / 食用植物 / 安全な収穫 / AI認識精度` はUI・データ・共有文・クエストから0件（CIで回帰検査）。
6. **薬効の非断定**：効能は「伝統的な用途・言い伝え」として表示し、医療目的利用の否定注記を付す。

## LLMの役割制限（実AI導入時 / §9.2F）
LLMは「検索済み情報の説明」に限定。禁止: 根拠のない薬効生成／存在しない特徴の追加／候補外の種の断定／安全性の独自判定／食用推奨。安全判定は必ずルールエンジン（公的情報: 厚労省・消費者庁）を優先。
